# MMLリファレンス

TL;DR: OnGen の MML は PPMCK/MCK 流の音長指定（`C4` = 四分音符の C）と相対オクターブ（`>` / `<`）を正本とします。
拍子は `#METER 4/4` のようなメタ情報として持ち、再生音ではなく解析・Lint・小節基準の確認に使います。

## 音長とオクターブ

音符直後の数字は**音長**を表します。オクターブは `O` コマンド、または `>`（1 つ上げる）/ `<`（1 つ下げる）で指定します。

```text
PPMCK / MCK: c4 = 四分音符のC
OnGen:       C4 = 四分音符のC（オクターブは O / > / < で指定）
```

旧バージョンでは音符直後の数字をオクターブとして解釈していましたが、現在は廃止されています。

## コマンド一覧

| コマンド | 意味 |
|---------|------|
| `C D E` | ノート（音長省略時は L コマンドの値） |
| `C4 D8 E16.` | ノート + 音長指定（付点も併用可） |
| `O4` | オクターブ指定 |
| `>` / `<` | オクターブを 1 つ上げる / 下げる |
| `L8` | デフォルト音長 |
| `T120` | テンポ (BPM) |
| `V15` | 音量 (0〜15) |
| `R` / `R8` | 休符 |
| `C&C` | スラー（同じ音程の音符を連結） |
| `[CDE]3` | ループ（任意回数の反復。数字省略時は 2 回） |
| `@1` | 音色プリセット（1〜7） |
| `@v0 = { 15,12,8 }` | 音量エンベロープ定義 |
| `@v0` | 音量エンベロープ適用 |
| `@EP0 = { 0,12 }` / `EP0` | ピッチエンベロープ定義・適用（半音単位、補間あり） |
| `@EN0 = { 0,4,7 }` / `EN0` | ノートエンベロープ定義・適用（半音単位、段階変化） |
| `@MP0 = { 6,5,12 }` / `MP0` | ピッチLFO定義・適用（delay, rate, depth） |
| `%12` | FM 変調指数 (0〜15) |
| `*4` | FM モジュレータ倍率（値÷2） |
| `~8` | LFO 深度 (0〜15) |
| `W(kick.wav)` | WAV サンプル再生 |
| `Q4` / `Q4,6` | ゲートタイム（音長の 4/8 を発音、第2引数は加算フレーム数） |
| `K+12` | トランスポーズ（半音） |
| `N32` / `N32,8` | 直接ノート番号（`n0` = C2、`n16` = C3、`n32` = C4）と音長 |
| `A`〜`E` | トラック宣言（1ファイル内の複数パート。行頭または `AB` 等の複数文字） |

独自拡張の `%`、`*`、`~`、`W(...)` は PPMCK/MCK 記法ではありません。

一致しないトークンは `ValueError` としてパーサーエラーになります（サイレントスキップしません）。

## 拍子メタ情報

MML本体は `T`、`L`、音符・休符の音長から再生時間を決める記法です。
そのため、拍子や小節は音を鳴らすための必須情報ではありません。

OnGenでは、輪唱の「1小節遅れ」や小節境界の検査を扱うために、拍子をメタ情報として記述できます。

```text
#TITLE Kaeru no Uta duet sample
#METER 4/4
A O4 L4 T120 C D E F
B O4 L4 T120 R1 C D E F
```

`#METER` は再生音を変えません。
`T120` かつ `#METER 4/4` の場合、1小節は四分音符4拍なので2秒です。
この基準により、`R1` を「1小節分の休符」として検証できます。

対応するメタ情報:

| メタ情報 | 意味 |
|----------|------|
| `#TITLE ...` | 曲名 |
| `#METER 4/4` | 拍子。`#TIME 4/4` も同義 |

拍子メタ情報は、解析・Lint・テストで小節基準を扱うための情報です。
拍子を省略したMMLは従来どおり再生できますが、「1小節遅れ」などの判定は人間側の仮定になります。

## PPMCK / MCK 互換性

| PPMCK / MCK記法 | 現状 | 備考 |
|-----------------|------|------|
| `c d e f g a b` | 部分対応 | 大文字・小文字を同一視 |
| `+`, `-` | 対応 | シャープ・フラット |
| `r4`, `r8` | 対応 | 休符の音長指定 |
| `t120` | 対応 | テンポ |
| `l4`, `l8` | 対応 | デフォルト音長 |
| `o4` | 対応 | オクターブ指定 |
| `v0`〜`v15` | 対応 | 固定音量 |
| `c4`, `c8` | 対応 | 音符ごとの音長指定 |
| `>`, `<` | 対応 | 相対オクターブ変更 |
| `c.`, `c4.` | 対応 | 音長指定と付点の組み合わせ |
| `&` | 対応 | 同じ音程の音符を連結 |
| `^` | 未対応 | タイの別表記（`&` を使用） |
| `q`, `Q` | 対応 | `q<num0>,<num1>`。音長 × `num0` / 分母 + `num1` フレーム。既定値は `q8,0`、分母は `#GATE-DENOM` で変更可 |
| `@q`, `@Q` | 対応 | フレーム単位の短縮（60fps 換算） |
| `[ ... ]4` | 対応 | `]` 直後の数字で反復回数 |
| `|: ... :|` | 未対応 | 多重リピート |
| `K<num>` | 対応 | 半音単位のトランスポーズ（-127〜126） |
| `n<num>,<len>` | 対応 | PPMCK 流の直接ノート番号。1オクターブを16値で表現し、12は未定義、13〜15は1オクターブ下のA〜B |
| トラック文字 `A`〜`E` | 対応 | 1ファイル内の複数トラック。曖昧な裸の音符列は単一トラックを優先。A/B=square、C=triangle、D/E=noise。`@0`〜`@3` は A/B のデューティ比 |
| `#TITLE` 等のヘッダ | 対応 | 行全体をスキップ（`;` 行コメントも同様） |
| `@v<num>` | 部分対応 | `@v<num> = { ... }` で音量エンベロープ定義、`@v<num>` で適用。`|` ループ指定に対応。値は60fps相当の点として扱い、クリックを避けるため線形補間 |
| `@EP<num>` | 部分対応 | `@EP<num> = { ... }` で定義、`EP<num>` で適用。値は半音単位のピッチ変化。60fps相当の点として扱い、線形補間 |
| `@EN<num>` | 部分対応 | `@EN<num> = { ... }` で定義、`EN<num>` で適用。値は半音単位の高速アルペジオ。60fps相当の点を段階変化として適用 |
| `@MP<num>` | 部分対応 | `@MP<num> = { delay, rate, depth }` で定義、`MP<num>` で適用。delay は60fps換算、rate はHz近似、depth は0〜15のOnGen LFO深度 |
| `@DPCM` | 未対応 | 代替として `W(...)` を使用 |

対応状況のロードマップは `_workingspace/plans/mml-compatibility-roadmap.md` にあります。Pyxel 方言は `_workingspace/plans/pyxel-mml-compatibility-plan.md` を参照してください。

## Pyxel 方言

TL;DR: `--mml-dialect pyxel` で Pyxel 2.4+ の単一 Sound 用 MML を解釈します。既定は `ppmck` です。

ゲーム連携の手順は [Pyxel 連携ガイド](pyxel-integration.md) を参照してください。

### CLI

```bash
python tools/sound/sfx_generator.py \
  --mml-dialect pyxel \
  --input-file scores/pyxel_core_sample.mml \
  -o output/pyxel/core_sample
```

### PPMCK との主な差分

| コマンド | PPMCK（既定） | Pyxel 方言 |
|---------|--------------|------------|
| `Q` | `Q4,6` 分数＋フレーム | `Q80` 発音率 0〜100％ |
| `V` | 0〜15 | 0〜127 |
| `&` | 同音程のみタイ | 同音=タイ、異音=レガート |
| `[...]` 省略 | 2 回 | `--max-repeat` 回（既定 2） |
| `@0`〜`@3` | A/B の duty 等 | triangle / square / pulse / noise |
| `Y` | 未対応 | デチューン（セント） |
| `#`（音符内） | `+` と同義 | `+` と同義 |
| `@ENV` `@VIB` `@GLI` | 未対応 | 簡易近似（完全一致非目標） |
| `A`〜`E` トラック | 対応 | 非対応（Sound 単位の MML） |

### フィクスチャ

`scores/pyxel_core_sample.mml` ほか `scores/pyxel_*.mml` を unittest で参照しています。

## 実行例

```bash
python tools/sound/sfx_generator.py --input "O4 L4 T120 C D E G" --style square -o output/mml/melody
```

ABC 記法は [BGMと楽譜](bgm-and-scores.md) を参照してください。

## よくある誤記

### `c5` はオクターブ5の C ではない

PPMCK MML では、音符直後の数字は**音長**です。

| 書き方 | 意味 |
|--------|------|
| `c5` | C note、音長 5（全音符の 1/5、T120 で 0.32s） |
| `> c4` | C note、四分音符、オクターブを 1 つ上げて演奏（C5） |
| `O5 c4` | C note、四分音符、オクターブ 5 を明示（C5） |

O4 文脈で C5 を鳴らすには `> c4 <` または `O5 c4 O4` と書きます。

`--structure-lint` を使うと `octave_notation_likely` で誤記の可能性を検出できます。

```bash
python tools/sound/sfx_generator.py --input-file scores/example.mml --structure-lint -o output/check/example
```

### `L2.` は使えない（L コマンドにドットは効かない）

`L` コマンドはデフォルト音長を数字で指定します。ドット修飾子（付点）は使えません。

```text
NG: L2.          ← パーサーエラーになる
OK: L4           ← デフォルトを四分音符にして、各ノートで C2. と書く
```

付点二分音符をデフォルトにしたい場合でも、ヘッダーは `L4`（または `L2`）のまま、個別のノートに付点を付けます。

### `T` コマンドは全トラックに書く

track A に書いた `T` コマンドは他のトラックに伝播しません。B〜E 各トラックのヘッダーにも同じ `T` を明記してください。

```text
NG:
A @1 V13 O4 L4 T120
...
B @2 V10 O3 L2        ← T が省略されると T120 デフォルトで動く

OK:
A @1 V13 O4 L4 T120
...
B @2 V10 O3 L2 T120   ← 全トラックに T を明記
```

`--structure-lint` で `track_tempo_mismatch` が出た場合、テンポを揃えていないトラックがあります。

### track B 内の `B-`（B フラット）を行頭に書かない

MML パーサーは行頭の `B` をトラックラベル（track B 宣言）として解釈しようとします。track B の内容として B♭（`B-`）音符を書く場合は、同一行の途中か、複数ノートを一行にまとめてください。

```text
NG（行頭に B- が来る場合）:
B @2 V9 O3 L4 T90
F2.
B-2.     ← 行頭の B がトラックラベルと誤解析される

OK（同一行にまとめる）:
B @2 V9 O3 L4 T90
F2. C2. B-2. C2.    ← B- が行中にあれば問題なし
```

### ループ体はバー境界に揃える

`[...]N` のループ本体が 1 小節に満たないと、`--structure-lint` で `loop_boundary_mismatch` が報告されます。ループ本体を 1 小節分（4/4 なら 8 分音符 8 つ、など）にまとめてください。

```text
NG:
[C8]128          ← ループ本体が 1 音符（1/8 小節）

OK:
[C8 C8 C8 C8 C8 C8 C8 C8]16   ← ループ本体 = 1 小節（4/4、8 分音符 8 つ）×16 回
```

3/4 拍子の場合、ループ本体は 6 個の 8 分音符（= 3 拍 = 1 小節）にします。

### `R2.`（付点休符）は使えない

ノートの付点（`C2.`）は動作しますが、休符の付点（`R2.`）はパーサーエラーになります。

```text
NG: R2.             ← パーサーエラー
OK: R2 R4           ← 二分休符 + 四分休符（= 3 拍分のタメ）
OK: R4 R4 R4        ← 四分休符 3 つ（3/4 の 1 小節休符）
```
